# UI/UX Design

## Design Principles

1. **Minimal & Focused** - Single purpose per view
2. **Dark Mode First** - Primary theme, light mode available
3. **Consistent Spacing** - 4px base unit (4, 8, 12, 16, 24, 32, 48)
4. **Clear Typography** - System fonts, readable sizes
5. **Subtle Animations** - Micro-interactions, not distracting

## Component Library

### Button

**Variants:**
- `primary` - Blue background, white text
- `secondary` - Transparent, border
- `ghost` - No background, hover shows
- `danger` - Red for destructive actions

**Sizes:**
- `sm` - Height 32px, padding 8px 12px
- `md` - Height 40px, padding 12px 16px
- `lg` - Height 48px, padding 16px 24px
- `fullWidth` - 100% width

**States:**
- Default, Hover, Active, Disabled, Loading

### Input

**Variants:**
- `default` - Bordered input
- `filled` - Background filled

**States:**
- Default, Focus (blue border), Error (red border), Disabled

### Card

```css
.card {
  background: var(--color-surface);
  border: 1px solid var(--color-border);
  border-radius: var(--radius-lg);
  padding: var(--spacing-xl);
}
```

### Toast Notifications

**Position:** Top-right corner, stacked
**Duration:** 5 seconds auto-dismiss
**Types:** success (green), error (red), warning (orange), info (blue)

## Page Specifications

### 1. Login Page (`/login`)

**Elements:**
- Logo + App name
- Tab switcher: Login | Register
- **Login Tab:**
  - Token input field
  - Show/hide toggle
  - Login button
- **Register Tab:**
  - Name input
  - Email input (read-only after registration)
  - Register button
- After registration: Display token with copy button

**States:**
- Default, Loading, Error (toast)

### 2. Revoke Token Page (`/revoke`)

**Route:** Protected — requires authenticated user

**Elements:**
- Logo + App name
- Warning heading: "Revoke Secret Key"
- Warning message: "This will log you out of all devices and generate a new secret key. The new key will be sent to your registered email."
- Current email display (read-only, from registration)
- "Revoke & Generate New Key" button
- "Cancel" link back to dashboard

**Flow:**
1. User clicks "Revoke & Generate New Key"
2. Client sends `POST /auth/revoke` with `Authorization: Bearer <current_token>`
3. Server updates secretKey in DB, sends new key to user's email
4. Client clears localStorage, redirects to `/login`
5. Toast: "Check your email for your new secret key"

**States:**
- Default, Loading, Error (toast)

### 3. Dashboard/Calendar Page (`/`)

**Elements:**
- Calendar header (month/week/day toggle, prev/next, today)
- Calendar grid (month view default)
- Task blocks on calendar days
- Event blocks with time
- Fixed task indicators
- Current time indicator

### 3. Timer Panel

**Position:** Fixed sidebar or floating panel

**Elements:**
- Current task name
- Circular progress indicator
- Time remaining (MM:SS)
- Session type indicator (Work/Short Break/Long Break)
- Controls: Start, Pause, Resume, Reset
- Skip break button (during breaks)

### 4. Task List

**Position:** Sidebar or below calendar

**Elements:**
- Task filters (status, category)
- Task cards:
  - Title
  - Duration indicator
  - Category color dot
  - Status badge
  - Pomodoro count
- Add task button

### 5. Reports Page (`/reports`)

**Elements:**
- Period selector (Weekly/Monthly/Yearly)
- Summary cards:
  - Total pomodoros
  - Total focus time
  - Tasks completed
  - Breaks taken
- Charts:
  - Bar chart (daily breakdown)
  - Line chart (trend)
  - Pie chart (category split)
  - Gauge (goal progress)

## Animations

### Transitions
```css
--transition-fast: 150ms ease;
--transition-base: 200ms ease;
--transition-slow: 300ms ease;
```

### Specific Animations
- Page transitions: Fade (200ms)
- Modal: Scale + Fade (200ms)
- Toast: Slide in from right (200ms)
- Button hover: Background color (150ms)
- Focus ring: Box shadow (150ms)

## Accessibility

- Focus indicators on all interactive elements
- ARIA labels for icons
- Keyboard navigation support
- Sufficient color contrast (4.5:1 minimum)
- Screen reader announcements for toasts
- Reduced motion preference respected
